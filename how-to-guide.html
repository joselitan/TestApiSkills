🚀 QA Test Management Platform

Komplett How-To Guide - Från Start till Dashboard

📋 Översikt: Denna guide visar dig exakt hur du startar hela QA Test Management Platform från början till slut. Du kommer att lära dig att starta servrar, köra tester, och använda både HTML och React dashboards.

1Starta Huvudapplikationen (Guestbook API)

Först måste vi starta huvudapplikationen som testerna kommer att köras mot.

cd src
python app.py
✅ Förväntat resultat:
* Running on http://127.0.0.1:8080
* Debug mode: on
Huvudapplikation körs på:
http://localhost:8080
⚠️ Viktigt: Låt denna terminal vara öppen! Applikationen måste köra för att testerna ska fungera.

2Starta Dashboard Backend

Nu startar vi dashboard backend som samlar in test-resultat och tillhandahåller AI-funktioner.

cd dashboard/backend
python app.py
✅ Förväntat resultat:
* Running on http://127.0.0.1:6001
* Debug mode: on
Dashboard Backend körs på:
http://localhost:6001
⚠️ Viktigt: Öppna en NY terminal för detta steg. Låt båda terminalerna vara öppna.

3Kör Tester (Real-time Dashboard Integration)

Nu kör vi tester som automatiskt skickar resultat till dashboard i real-time.

💡 Tips: Öppna dashboard:en INNAN du kör testerna så ser du real-time uppdateringar!

Kör UI-tester:

cd tests/ui
pytest test_guestbook_ui.py -v

Kör API-tester:

cd tests/api
pytest test_guestbook_api.py -v

Kör Performance-tester:

cd tests/performance
pytest test_load_testing.py -v
✅ Vad händer:
• Varje test skickas till dashboard när det slutförs
• AI analyserar misslyckade tester automatiskt
• Test-typ detekteras automatiskt (ui/api/performance)
• Dashboard uppdateras i real-time
⚠️ Observera: Du kommer att se meddelanden som:
"✅ test_name -> Dashboard uppdaterad"
"🤖 AI skapade X förbättringsförslag!"

6Komplett Test Management Workflow

Nu när allt är igång, här är det kompletta workflow:et för test management.

🔄 Daglig Workflow:

  1. Starta servrar (steg 1-2)
  2. Öppna dashboard (HTML eller React)
  3. Kör tester - se real-time uppdateringar
  4. Granska AI-förslag för misslyckade tester
  5. Skapa bug reports för kritiska issues
  6. Hantera test cases via React dashboard

🤖 AI-Powered Features:

  • Failure Analysis - AI analyserar misslyckade tester
  • Code Analysis - Klistra in kod för test-förslag
  • Pattern Detection - Upptäcker flaky/slow tests
  • Smart Suggestions - Confidence-baserade rekommendationer

🔗 Integrationer:

  • GitHub Issues - Auto-skapa issues från bug reports
  • Jira Tickets - Koppla bugs till Jira
  • Slack Notifications - Automatiska meddelanden

🔧 Troubleshooting

Problem: "Dashboard visar ingen data"

  • Kontrollera att backend körs på port 6001
  • Testa: curl http://localhost:6001/api/dashboard/enhanced-summary
  • Klicka på "🔄 Uppdatera" knappen

Problem: "React appen kan inte nå backend"

  • Kontrollera att båda servrarna körs (8080 och 6001)
  • Starta om React appen: npm start
  • Kontrollera browser console för fel

Problem: "Tester skickas inte till dashboard"

  • Kontrollera att dashboard backend körs
  • Kör tester från projekt root-mappen
  • Se efter meddelanden: "Dashboard uppdaterad"

Problem: "AI-förslag visas inte"

  • Kör tester som misslyckas för att generera AI-förslag
  • Kontrollera AI Insights tab i React dashboard
  • Testa: curl http://localhost:6001/api/ai/suggestions
