📋 Översikt

Genomförd Förbättring ✅

Implementerat separata databaser för test och produktion enligt QA Platform Roadmap - "Vad Vi Kan Förbättra Direkt" punkt 2.

Problem

Innan: Testdata och produktionsdata blandades i samma databas (guestbook.db)
  • ❌ Risk att tester förstör produktionsdata
  • ❌ Svårt att rensa testdata utan att påverka produktion
  • ❌ Kan inte köra tester parallellt med produktion
  • ❌ Testdata synlig i produktions-UI

Lösning

Nu: Separata databaser med automatisk hantering
  • ✅ guestbook.db - Produktionsdatabas (app.py)
  • ✅ guestbook_test.db - Test-databas (tester)
  • ✅ Automatisk setup via pytest fixtures
  • ✅ Enkel cleanup av testdata
  • ✅ Säker testning utan risk för produktionsdata

🔧 Implementerade Ändringar

1. database.py - Uppdaterad med test_mode

c:\Users\admin\TestApiSkills\database.py
Ändringar: Lagt till test_mode parameter i alla databas-funktioner
# FÖRE:
def get_db_path():
    basedir = os.path.abspath(os.path.dirname(__file__))
    return os.path.join(basedir, 'guestbook.db')

# EFTER:
def get_db_path(test_mode=False):
    basedir = os.path.abspath(os.path.dirname(__file__))
    db_name = 'guestbook_test.db' if test_mode else 'guestbook.db'
    return os.path.join(basedir, db_name)
Uppdaterade funktioner:
  • get_db_path(test_mode=False) - Returnerar rätt databas-sökväg
  • init_db(test_mode=False) - Initierar rätt databas
  • get_db(test_mode=False) - Ansluter till rätt databas
Ny funktion:
def clear_test_db():
    """Clear all data from test database"""
    test_db_path = get_db_path(test_mode=True)
    if os.path.exists(test_db_path):
        conn = sqlite3.connect(test_db_path)
        cursor = conn.cursor()
        cursor.execute("DELETE FROM guestbook")
        conn.commit()
        conn.close()
        return True
    return False

2. conftest.py - Pytest Fixtures (NY FIL)

c:\Users\admin\TestApiSkills\conftest.py
Syfte: Automatisk setup och hantering av test-databas
import pytest
import os
from database import init_db, clear_test_db, get_db_path

os.environ['TEST_MODE'] = '1'

@pytest.fixture(scope="session", autouse=True)
def setup_test_database():
    """Setup test database before all tests"""
    print("\n🔧 Setting up test database...")
    init_db(test_mode=True)
    print(f"✅ Test database created: {get_db_path(test_mode=True)}")
    yield

@pytest.fixture(scope="function")
def clean_test_db():
    """Clean test database before each test"""
    clear_test_db()
    yield

@pytest.fixture(scope="session")
def test_db_path():
    """Return test database path"""
    return get_db_path(test_mode=True)
Fixtures:
  • setup_test_database: Körs automatiskt innan alla tester (session scope)
  • clean_test_db: Kan användas för att rensa DB innan specifika tester
  • test_db_path: Returnerar sökväg till test-databasen

🚀 Hur Man Använder Det

Scenario 1: Kör Produktion (Normal App)

# Startar produktion med guestbook.db
python app.py

# Öppna http://localhost:8080
# All data sparas i guestbook.db

Scenario 2: Kör Automatiska Tester

# Tester använder automatiskt guestbook_test.db
pytest test_guestbook_api.py -v

# Med HTML rapport och coverage
pytest test_guestbook_api.py --html=reports/test_report.html --cov=. --cov-report=html

# Utan cleanup (data stannar för inspektion)
pytest test_guestbook_api.py -k "not zzz_cleanup" -v

Scenario 3: Manuell Testning med Test App

# Starta test app (använder guestbook_test.db)
python test_app.py

# Öppna http://localhost:8080
# All data sparas i guestbook_test.db
# Produktionsdata påverkas inte

Scenario 4: Rensa Test Database

# Alternativ 1: Via Python
python -c "from database import clear_test_db; clear_test_db()"

# Alternativ 2: Via pytest cleanup test
pytest test_guestbook_api.py -k "zzz_cleanup" -v

# Alternativ 3: Ta bort filen
del guestbook_test.db # Windows
rm guestbook_test.db # Mac/Linux

✨ Fördelar med Denna Lösning

Säkerhet & Isolation

  • ✅ Dataskydd: Tester kan aldrig förstöra produktionsdata
  • ✅ Isolation: Test och produktion är helt separerade

  • ✅ Automatisk setup: Pytest fixtures hanterar allt
  • ✅ Enkel cleanup: En funktion rensar all testdata

CI/CD Vänlig

  • ✅ Automatisk: Fungerar direkt i GitHub Actions
  • ✅ Ren miljö: Varje test-körning får en ren databas
  • ✅ Ingen konfiguration: Fungerar out-of-the-box
