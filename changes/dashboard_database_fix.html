🛠️ Dashboard: Databasfilschema fix

Problem med att testresultat inte sparas i dashboard-databasen

Senast uppdaterad: 2026-03-16

❗ Problem: Dashboard visar inga testresultat

När tester körs så postas resultat till http://localhost:6001/api/test-runs, men dashboard-API:t svarade fortfarande med 0 i statistiken. Felet visade sig i loggen som ett SQLite-schema-fel:
sqlite3.OperationalError: table test_cases has no column named test_name
Det betyder att databasen som Flask-servern använder har en felaktig version av test_cases-tabellen.

🔍 Orsak: Felaktigt / föråldrat DB-schema

Dashboard-backend kördes mot en SQLite-fil som saknade de kolumner som koden försökte skriva i (t.ex. test_name, run_id etc). Detta kan hända om:
  • Dashboard startas från fel katalog -> skapar "/dashboard/database/test_dashboard.db" i en annan plats
  • En gammal databasfil från tidigare version av komplex dashboard-logik ligger kvar
  • Schema-ändringar (nya kolumner) införts utan migrering

✅ Löst: Schema-migrering + säker start

Följande fix lades till i dashboard/backend/app.py:
# Efter att tabellen skapats, kontrollera att den har alla kolumner som API:t använder cursor.execute("PRAGMA table_info(test_cases)") columns = {row[1] for row in cursor.fetchall()} required_columns = { "run_id": "INTEGER", "test_name": "TEXT", "test_file": "TEXT", "status": "TEXT", "duration": "REAL", "error_message": "TEXT", } for col, col_type in required_columns.items(): if col not in columns: cursor.execute(f"ALTER TABLE test_cases ADD COLUMN {col} {col_type}")
Denna migrering gör databasen bakåtkompatibel så att äldre filer automatiskt får rätt schema när servern startar.

✅ Verifiering

Efter fixen validerades att datan faktiskt skrivs till databasen:
  • Kör ett test som postar till /api/test-runs
  • Kontrollera att följande API ger icke-nollvärden: http://localhost:6001/api/dashboard/summary
  • Bekräfta att dashboard/database/test_dashboard.db innehåller rader i test_runs

🧩 Tips - Starta alltid från projektroten

För att undvika att skapa flera olika DB-filer (på grund av olika arbetskatalog) bör du alltid köra:
cd c:\Users\admin\TestApiSkills python dashboard/backend/app.py

📌 Slutsats

Dashboarden är nu fullt funktionell; testresultat sparas i den avsedda databasen och UI:t visar data korrekt.
Den här dokumentationen finns för att hjälpa framtida underhållare att snabbt förstå varför databasen behövde migreras och hur man verifierar att allt fungerar.
