1. app.py - Swagger Konfiguration

c:\Users\admin\TestApiSkills\app.py
Tillagt: Flasgger import och konfiguration
from flasgger import Swagger

# Swagger configuration
swagger_config = {
    "headers": [],
    "specs": [{
        "endpoint": 'apispec',
        "route": '/apispec.json',
    }],
    "swagger_ui": True,
    "specs_route": "/apidocs/"
}

swagger_template = {
    "swagger": "2.0",
    "info": {
        "title": "Guestbook API",
        "description": "RESTful API for Guestbook application",
        "version": "1.0.0"
    },
    "securityDefinitions": {
        "Bearer": {
            "type": "apiKey",
            "name": "Authorization",
            "in": "header"
        }
    }
}

swagger = Swagger(app, config=swagger_config, template=swagger_template)
Resultat:
  • Swagger UI tillgänglig på /apidocs/
  • OpenAPI spec på /apispec.json
  • JWT Bearer authentication konfigurerad

3. Endpoint Dokumentation - Guestbook CRUD

Alla endpoints dokumenterade:
  • POST /api/guestbook - Skapa ny entry
    • Request body schema med required fields
    • Exempel-värden för alla fält
    • Response schemas för 201, 400, 401
  • GET /api/guestbook - Hämta alla entries
    • Query parameters: page, limit, search
    • Pagination metadata i response
    • Array av entry-objekt
  • GET /api/guestbook/{id} - Hämta en entry
    • Path parameter: user_id
    • Response för 200 och 404
  • PUT /api/guestbook/{id} - Uppdatera entry
    • Path parameter + request body
    • Required fields dokumenterade
  • DELETE /api/guestbook/{id} - Radera entry
    • Path parameter
    • Success/error responses
  • DELETE /api/guestbook/bulk - Bulk delete
    • Array av IDs i request body
    • Antal raderade i response
  • POST /api/guestbook/import - Excel import
    • File upload (multipart/form-data)
    • Excel format requirements

🚀 Hur Man Använder Swagger UI

Steg 1: Starta Applikationen

python app.py

Steg 2: Öppna Swagger UI

Gå till: http://localhost:8080/apidocs/
Du ser nu en interaktiv API-dokumentation med:
  • 📁 Authentication - Login endpoint
  • 📁 Guestbook - Alla CRUD endpoints

Steg 3: Logga In och Få Token

  1. Klicka på POST /api/login
  2. Klicka "Try it out"
  3. Fyll i credentials:
    {
      "username": "admin",
      "password": "password123"
    }
  4. Klicka "Execute"
  5. Kopiera token från response

Steg 4: Auktorisera

  1. Klicka på "Authorize" knappen (🔒 högst upp)
  2. Skriv: Bearer {din-token-här}
  3. Klicka "Authorize"
  4. Klicka "Close"
Nu är du auktoriserad och kan testa alla endpoints! 🎉

Steg 5: Testa Endpoints

Exempel: Skapa en guestbook entry
  1. Klicka på POST /api/guestbook
  2. Klicka "Try it out"
  3. Fyll i data (exempel finns redan)
  4. Klicka "Execute"
  5. Se request och response direkt!
