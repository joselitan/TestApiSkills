🖥️Systemkrav

Minimum Requirements

  • OS: Windows 10/11, macOS 10.15+, Ubuntu 18.04+
  • Python: 3.11 eller senare
  • RAM: 4GB (8GB rekommenderat)
  • Disk: 2GB ledigt utrymme
  • Internet: Krävs för installation och CI/CD

Verktyg som behövs

  • Git - För versionshantering
  • Docker (valfritt) - För containerized deployment
  • Web browser - Chrome/Firefox/Edge för UI-tester

📦Installation

1Klona Repository

git clone https://github.com/yourusername/qa-test-platform.git
cd qa-test-platform

2Skapa Virtual Environment

# Windows
python -m venv venv
venv\Scripts\activate

# macOS/Linux  
python3 -m venv venv
source venv/bin/activate

3Installera Dependencies

pip install --upgrade pip
pip install -r requirements.txt

4Installera Playwright Browsers

playwright install chromium

5Verifiera Installation

python scripts/check_setup.py
✅ Om allt är OK ser du: 
✓ Python version: 3.11.x
✓ All dependencies installed
✓ Database connection: OK
✓ Playwright browsers: OK
✓ Installation complete!

Integrationstester (Webhook System)

Dessa tester verifierar att plattformen kan kommunicera med externa system via webhooks. En lokal mock-server startas automatiskt — inga externa beroenden behövs.

pytest tests/api/test_integrations.py -v
Vad integrationstesterna täcker:
  • Registrera och avregistrera webhook-prenumeranter
  • Lista aktiva prenumeranter
  • Ta emot inkommande händelser från externa system
  • End-to-end: en gästbokspost skapas och webhook-anropet fångas upp av mock-servern
  • HMAC-SHA256 signaturverifiering

Registrera en webhook manuellt (valfritt)

# Hämta ett token först
TOKEN=$(curl -s -X POST http://localhost:8080/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"password123"}' | python3 -c "import sys,json; print(json.load(sys.stdin)['token'])")

# Registrera en webhook
curl -X POST http://localhost:8080/api/webhooks \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://ditt-system.example.com/hook", "events": ["entry.created", "user.registered"]}'

🔄CI/CD Setup

GitHub Repository Setup

1Skapa GitHub Repository

git remote add origin https://github.com/yourusername/your-qa-project.git
git push -u origin main

2Konfigurera GitHub Secrets

  • Gå till: Settings → Secrets and variables → Actions
  • Lägg till secrets (om du har externa services)

3Aktivera GitHub Actions

  • CI/CD pipeline startar automatiskt vid push
  • Se status under "Actions" tab

Pipeline Stages

  1. 🔍 Code Quality - Black, flake8, pylint
  2. 🧪 Unit Tests - Grundläggande tester
  3. 🔌 API Tests - Integration tester
  4. 🔒 Security Scan - Säkerhetsanalys
  5. 🖥️ UI Tests - Browser automation
  6. ⚡ Performance Tests - Load testing
  7. 🐳 Build - Docker image
  8. 🚀 Deploy - Staging & Production
