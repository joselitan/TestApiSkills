3.1 OWASP Top 10 Testing

Mål:

Testa mot de vanligaste säkerhetshoten enligt OWASP

Test Cases:

  • A01 - Broken Access Control: Testa authorization
  • A02 - Cryptographic Failures: Verifiera JWT, password hashing
  • A03 - Injection: SQL injection, XSS, command injection
  • A04 - Insecure Design: Rate limiting, brute force protection
  • A05 - Security Misconfiguration: Debug mode, error messages
  • A06 - Vulnerable Components: Dependency scanning
  • A07 - Authentication Failures: Weak passwords, session management
  • A08 - Data Integrity: Input validation
  • A09 - Logging Failures: Security event logging
  • A10 - SSRF: Server-side request forgery

Nya Filer:

  • Metrics: Response time, throughput, error rate
  • Real-time Dashboard: Web UI för monitoring

Ny Fil:

tests/performance/locustfile.py

from locust import HttpUser, task, between

class GuestbookUser(HttpUser):
    wait_time = between(1, 3)
    
    def on_start(self):
        # Login och få token
        response = self.client.post("/api/login", json={...})
        self.token = response.json()["token"]
    
    @task(3)
    def read_entries(self):
        self.client.get("/api/guestbook", headers={...})
    
    @task(1)
    def create_entry(self):
        self.client.post("/api/guestbook", json={...}, headers={...})

# Kör:
locust -f tests/performance/locustfile.py --users 100 --spawn-rate 10

🔄 Fas 5: CI/CD Enhancement PLANERAD

⏱️ Tidsåtgång: 1-2 veckor | Prioritet: MEDEL

5.1 Multi-Stage Pipeline

Mål:

Skapa en komplett CI/CD pipeline med flera stages

Pipeline Stages:

  1. Lint & Format: Black, flake8, pylint
  2. Unit Tests: Snabba tester
  3. Integration Tests: API tester
  4. Security Scan: Bandit, Safety
  5. UI Tests: Playwright E2E
  6. Performance Tests: Load testing
  7. Build: Docker image
  8. Deploy: Till staging/production

Uppdaterad Fil:

.github/workflows/ci-cd-pipeline.yml

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - run: black --check .
      - run: flake8 .

  unit-tests:
    needs: lint
    runs-on: ubuntu-latest
    steps:
      - run: pytest tests/unit/ -v

  api-tests:
    needs: unit-tests
    runs-on: ubuntu-latest
    steps:
      - run: pytest test_guestbook_api.py --html=report.html

  security-scan:
    needs: api-tests
    runs-on: ubuntu-latest
    steps:
      - run: bandit -r . -f json
      - run: safety check

  ui-tests:
    needs: api-tests
    runs-on: ubuntu-latest
    steps:
      - run: playwright install
      - run: pytest tests/ui/ -v
